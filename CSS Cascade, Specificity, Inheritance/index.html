Specificity

Specificity is the way browser decides which CSS property values are the most relevant to an element and gets applied.based on the selector

Higher will be the Specificity of selectors, more importance will be given to him.

CSS specificity determines which style rules get applied to an element when there are conflicts.

Different Selectors have different Specificity

Inline Style > ID Selector > Class or Attribute Selector > Element Selector > Universal Selector
  • Universal Selector: 0
  • Element selectors and pseudo-elements: 1
  • Class selectors, attribute selectors, and pseudo-classes: 10
  • ID selectors: 100
  • Inline styles: 1000
        p {
            color: red !important;
          }
          
        p {
            color: blue;
          }

!important keyword help to increase specificity

We can increase Specificity by Repeat Multiple Selectors

    .p.p.p.p.p{
        color: red;
      }

    .p{
        color: blue;
      }

We can also over write css which have !important keyword this way

      .p.p{
        color: blue !important;
      }

      .p{
        color: red !important;
      }

Cascades

If make conflict with same specificity selectors then, the last one will be applied

means If we apply similar CSS property to single element with same selector then(same specificity),then whatever the last changes we made on, it will be applied

because Java script engine parses CSS from top to bottom that's why last changes will be applied every time,if selectors have same specificity, similar property gets overwritten by last changes

Inheritance

Inheritance is the mechanism by which properties are applied not only to a specified element, but also to its descendants.

Properties are inherited from the parent element to the child element.
